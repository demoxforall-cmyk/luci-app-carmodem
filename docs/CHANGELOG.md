# CHANGELOG

История версий luci-app-carmodem. Версии — `PKG_VERSION` в `luci-app-carmodem/Makefile`.

## 3.8 — производительность (быстрые победы, по итогам аудита)
- **AT-обмен ускорен**: опрос ответа модема шагом **0.1 c вместо 1 c** (BusyBox fancy sleep; детект кэшируется, безопасный фолбэк на 1 c). Латентность каждой AT-команды падает с ≥1 c до ~0.1–0.3 c, AT-порт (телеметрия/консоль/CPMS-счётчики) освобождается в разы быстрее.
- **Журнал дозвона**: `logread -l 500` — не выгружаем и не грепаем весь кольцевой буфер syslog каждые 2 c (важно при увеличенном `log_size`).
- **Реестр отправленных SMS**: запись и удаление под общим `flock` (закрыта гонка append↔delete), уникальный tmp (без затирания при параллельном удалении), **ротация последних 500 записей** (страховка от роста файла в /etc).
- **Меньше форков в горячих функциях**: `cm_mm_conn` (тик 1 c на Advanced) — один awk-проход вместо 2×(sed|head); `cm_mm_status` читает счётчики трафика встроенным `read` вместо `cat`.
- **Багфикс (был с 3.7):** удаление всех/последней записи из реестра отправленных не срабатывало — `grep -v` при пустом результате давал `rc=1` и отменял перезапись файла, «удалённое» исходящее возвращалось после обновления. Теперь удаление безусловно, учёт `deleted/failed` точный, а сбой `cp` не обнуляет реестр.

## 3.7 — багфиксы (по итогам аудита кода)
- **Band-lock заработал**: UI-токены `B1,n78` теперь конвертируются в имена ModemManager с правильным разделителем (`eutran-1|ngran-78`); пустой список = снять блокировку (`any`). Раньше кнопка «Save & apply» всегда возвращала ошибку.
- **RAT-режим «Авто» заработал**: `auto` маппится в `any` для mmcli; whitelist режимов ужесточён (раньше `case *g` пропускал любую строку на «g»).
- **SIM-слот и TTL реализованы** (были заглушками с `not_implemented`): слот — через `AT+QUIMSLOT` (RM520N-GL), TTL — через nftables-инклюд fw4 (`/etc/nftables.d/90-carmodem-ttl.nft`, in/out, hoplimit для IPv6, применение с проверкой `fw4 check`, 0/0 = снять).
- **Индекс модема резолвится динамически** (`mmcli -L` раз на rpcd-вызов): после Reset modem / смены USB-режима / слота MM даёт модему новый индекс — раньше весь бэкенд слеп до перезагрузки. Плюс самолечение `--signal-setup` после переиндексации.
- **SMS с апострофом отправляются**: `'` в тексте ломал key-value парсер mmcli (и позволял дописать чужие свойства SMS) — теперь номер строго валидируется, апостроф заменяется типографским U+2019.
- **Уникальные id в реестре отправленных** (`s<epoch>_<sid>`): голый sid MM обнуляется при каждом ребуте, дубль id ломал выборочное удаление.
- **Submit-объекты чистятся из MM** сразу после успешной отправки — раньше копились до ребута и раздували каждый полный `get_sms`.
- **Band-lock атомарен**: применение любой карточки шлёт объединённый выбор LTE+NR (`SetCurrentBands` у MM заменяет весь список — раньше применение LTE стирало NR-лок, риск потери связи).
- **TTL-фикс предупреждает про flow offloading**: если включён — оффлоаженные соединения обходят правило, UI показывает подсказку отключить «Software flow offloading».
- Защита от инъекции в номере SMS усилена: перевод строки в поле номера больше не обходит валидацию (grep построчный → добавлен `case`-гард).

Известные ограничения 3.7: (1) апостроф в **латинском** SMS заменяется на «’» (U+2019) — такое сообщение кодируется UCS-2 (для кириллицы неважно — она и так UCS-2); (2) TTL-правило привязано к имени wwan-интерфейса на момент установки — после смены USB-режима модема правило стоит применить заново.

## 3.6
- Messages: **входящие SMS появляются сами** — живое обновление без перезагрузки страницы. Часто опрашивается дешёвый сигнал (1 вызов mmcli — `get_sms_sig`), полная загрузка списка делается только при изменении. Набираемый текст, фокус и позиция прокрутки сохраняются; при чтении истории — плашка «новые ↓» и счётчик непрочитанных в списке.

## 3.5
- Messages: **исходящие SMS больше не пропадают**. RM520N не хранит копию отправленного — теперь ведём собственный постоянный реестр (`/etc/carmodem/sent_sms.jsonl`: номер+текст+время), и тред показывает исходящие из него, независимо от модема. Дублей нет (submit-копии модема пропускаются, id реестра — в отдельном пространстве `sNN`).
- Messages: сортировка треда — по времени (а не по id модема, который не отражает хронологию).

## 3.4
- Status: в таблицах Carrier aggregation и Neighbour cells столбцы данных равной ширины (промежуток Type↔Band = остальным); ★-столбец не трогается.
- Messages: панель контактов сужена 38% → 34% (больше места переписке).
- Advanced / AT-консоль: добавлен реестр команд смены интерфейса RM520N — `AT+QCFG="usbnet"` (QMI/ECM/MBIM/RNDIS/NCM), `AT+QCFG="data_interface"` (USB/PCIe), `AT+CFUN=1,1`.

## 3.3
- Advanced / Connection: кнопка **живого журнала дозвона** — фрейм выше band-lock с логом ModemManager/netifd/wwan в реальном времени (бэкенд `get_conn_log`), повторное нажатие убирает.

## 3.2

## 2.4

## 2.0–2.3
